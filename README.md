# Оценка временной сложности алгоритмов (algochecker)

Программный комплекс для статической и динамической оценки временной сложности алгоритмов, написанных на C++, с поддержкой веб-интерфейса и визуализации графиков.

## Структура проекта

```text
algochecker/
├── CMakeLists.txt                  # Скрипт сборки проекта
├── README.md                       # Документация (этот файл)
├── src/                            # Исходные файлы (.cpp)
│   ├── main.cpp                    # Точка входа (CLI и инициализация веб-сервера)
│   ├── static_analyzer/            # Модуль статического анализа
│   │   ├── file_loader.cpp
│   │   ├── lexer.cpp
│   │   ├── parser.cpp
│   │   └── complexity_evaluator.cpp
│   ├── dynamic_analyzer/           # Модуль бенчмаркинга
│   │   ├── data_generator.cpp
│   │   ├── benchmarker.cpp
│   │   ├── exporter.cpp
│   │   └── visualizer.cpp
│   └── web_server/                 # HTTP сервер для связи с фронтендом
│       └── server.cpp
├── include/                        # Заголовочные файлы (.h, .hpp), повторяют структуру src/
│   ├── static_analyzer/
│   ├── dynamic_analyzer/
│   └── web_server/
├── web/                            # Статические файлы для веб-интерфейса
│   ├── index.html
│   ├── style.css
│   └── script.js
├── tests/                          # Тестовые примеры кода для анализа (.txt, .cpp)
└── scripts/                        # Скрипты (.gp) для Gnuplot
```

## Распределение ролей и задач (План разработки)

### Этап 1: Проектирование и базовый сетап

- Все разработчики работают совместно.
- Матвеев: Настройка репозитория, написание базового `CMakeLists.txt`, создание заглушек в папках `src/` и `include/`.
- Леонов: Проектирование структур данных (классов) для хранения результатов парсинга.
- Новицкий: Создание драфта `web/index.html` и `web/style.css`.

### Этап 2: Активная разработка модулей

#### Леонов: Модуль статического анализа

- [x] ФТ-1.1 Загрузка исходного кода. Файл: `src/static_analyzer/file_loader.cpp` (логика чтения `.txt` и `.cpp` в буфер).
- [x] ФТ-1.2 Лексический разбор и фильтрация. Файл: `src/static_analyzer/lexer.cpp` (очистка от `//`, `/* */` и строковых литералов `""`).
- [x] ФТ-1.3 Идентификация управляющих конструкций. Файл: `src/static_analyzer/parser.cpp` (поиск `for`, `while`, `do` и границ `{ }`).
- [x] ФТ-1.4 Расчет глубины вложенности циклов. Файл: `src/static_analyzer/parser.cpp` (счетчик уровней внутри найденных блоков).
- [x] ФТ-1.5 Подсчет элементарных операций. Файл: `src/static_analyzer/parser.cpp` или `src/static_analyzer/operation_counter.cpp` (поиск арифметики, присваиваний и т.д. внутри циклов).
- [x] ФТ-1.6 Расчет теоретической функции сложности. Файл: `src/static_analyzer/complexity_evaluator.cpp` (формирование `T(n)` и аппроксимация до `O(...)`).

#### Матвеев: Модуль динамического анализа (бенчмаркинг)

- [ ] ФТ-2.1 Генерация тестовых данных. Файл: `src/dynamic_analyzer/data_generator.cpp` (генерация случайных массивов/векторов на 100, 1000, 5000, 10000 элементов).
- [ ] ФТ-2.2 Замер времени выполнения. Файл: `src/dynamic_analyzer/benchmarker.cpp` (использование `std::chrono` для запуска тестов).
- [ ] ФТ-2.3 Сбор и экспорт статистики. Файл: `src/dynamic_analyzer/exporter.cpp` (запись `T(N)` в `.csv` или `.dat`).
- [ ] ФТ-2.4 Визуализация результатов. Файл: `src/dynamic_analyzer/visualizer.cpp` (генерация `.gp` файлов и вызов Gnuplot через `system()`).

#### Новицкий: Full-Stack интеграция и веб-интерфейс

- [ ] CLI-интерфейс (часть ФТ-1.1). Файл: `src/main.cpp` (обработка аргументов командной строки как резервный вариант работы).
- [ ] Веб-сервер / API. Файл: `src/web_server/server.cpp` (интеграция cpp-httplib, роутинг, прием кода через POST-запрос и отдача результатов).
- [ ] Фронтенд: структура и стили. Файлы: `web/index.html`, `web/style.css` (верстка формы ввода, кнопок и блока с результатами).
- [ ] Фронтенд: логика. Файл: `web/script.js` (Fetch API для отправки введенного кода на сервер и отрисовки полученной функции Big-O и картинки с графиком).

### Этап 3: Объединение (сборка)

- Точка синхронизации работы всех участников.
- Новицкий: подключает заголовки `complexity_evaluator.h` (от Разр. 1) и `visualizer.h` (от Разр. 2) в `src/web_server/server.cpp`.
- Матвеев: настраивают корректные пути сохранения картинок Gnuplot, чтобы веб-сервер мог отдавать их фронтенду.
- Леонов: формирует папку `tests/` с эталонными алгоритмами для проверки всей цепочки.

### Этап 4: Тестирование и обработка ограничений

- Все разработчики проверяют систему на соответствие Ограничениям 1, 2 и 3 (упрощенный синтаксис, игнорирование рекурсии, линейная зависимость).
- Отладка парсера на некорректном коде.
- Обработка ошибок в `script.js` (например, вывод уведомления "Ошибка анализа кода" вместо падения приложения).
